---
title: "Quickstart"
description: "Get Peermetrics running and connected to your WebRTC app in minutes"
---

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed:

    - [Docker](https://docs.docker.com/get-docker/) 20.10 or later
    - [Docker Compose](https://docs.docker.com/compose/install/) v2 or later (included with Docker Desktop)

    Verify both are available:

    ```bash
    docker --version
    docker compose version
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/peermetrics/peermetrics
    cd peermetrics
    ```
  </Step>

  <Step title="Start with Docker Compose">
    Start all services — nginx, API, web, PostgreSQL, and Redis — with a single command:

    ```bash
    docker compose up
    ```

    On first startup, the API service automatically runs database migrations and creates the default admin user.

    <Note>
      For a more reliable local setup that builds from source and includes all migrations, use the dev compose file instead:

      ```bash
      # Clone the api and web repos in the same parent directory first
      git clone https://github.com/peermetrics/api
      git clone https://github.com/peermetrics/web
      cd peermetrics

      docker compose -f docker-compose.dev.yaml up
      ```

      This is the recommended approach for development.
    </Note>
  </Step>

  <Step title="Access the web dashboard">
    Once all containers are running, open the dashboard in your browser:

    **[http://localhost:8080](http://localhost:8080)**

    Log in with the default credentials:

    - **Username**: `admin`
    - **Password**: `admin`

    <Warning>
      The default `admin` / `admin` credentials must be changed before any production deployment. Peermetrics will prompt you to change the password on first login. You can also set custom credentials via environment variables — see the [Docker deployment guide](/deployment/docker) for details.
    </Warning>
  </Step>

  <Step title="Create an organization, app, and API key">
    Inside the dashboard:

    1. Create a new **organization**.
    2. Inside that organization, create a new **app** — this represents your WebRTC application.
    3. Copy the **API key** shown on the app dashboard. You'll need it in the next step.
  </Step>

  <Step title="Integrate the JavaScript SDK">
    Install the Peermetrics JavaScript SDK in your WebRTC application:

    ```bash
    npm install @peermetrics/sdk
    ```

    Initialize it with the `apiRoot` option pointing to the local API service, using the API key you copied in the previous step:

    ```javascript
    let peerMetrics = new PeerMetrics({
        apiRoot: 'https://api.example.com/v1',
        apiKey: '7090df95cd247f4aa735779636b202', // api key from the newly created app
        userId: '1234',
        userName: 'My user',
        conferenceId: 'conference-1',
        conferenceName: 'Conference from 4pm',
        appVersion: '1.0.1'
    })
    ```

    When running locally, set `apiRoot` to `http://localhost:8081/v1`.

    <Warning>
      `apiRoot` must be a valid URL and must end with `/v1`.
    </Warning>

    For the full SDK integration guide including framework-specific setup for Livekit, Mediasoup, Janus, and custom WebRTC implementations, see [JavaScript SDK](/integration/javascript-sdk).
  </Step>

  <Step title="View metrics in the dashboard">
    Once your WebRTC app is running with the SDK integrated:

    1. Start or join a call in your application.
    2. Return to the dashboard at **[http://localhost:8080](http://localhost:8080)**.
    3. Navigate to your app to see conferences, participants, and connection metrics.

    You can also use the built-in test tool to verify your setup without modifying your app. Navigate to **[http://localhost:8080/test/webrtc-test.html](http://localhost:8080/test/webrtc-test.html)**, enter your API key and `http://localhost:8081/v1` as the API root, then click **Start Test Call**.
  </Step>
</Steps>
